## Security checks performed

- Confirmed public issue content is parsed only as YAML data by the GitHub Action.
- Confirmed no workflow uses `pull_request_target`.
- Confirmed CI workflow uses minimal `contents: read` permission.
- Confirmed generated issue workflow uses `issues: write` and `contents: read` only.
- Confirmed the GitHub Action does not execute manifest commands, does not call external APIs other than GitHub issue APIs, and supports dry-run mode.
- Confirmed GitHub Action comments are marker-based to avoid duplicates.
- Added comment-text sanitization for Action-generated comments.
- Confirmed safe-run is opt-in and prints Docker commands before execution.
- Confirmed safe-run uses Docker argument arrays rather than host-shell command strings.
- Confirmed safe-run command construction includes `--network none`, non-root `--user 1000:1000`, `--read-only`, read-only repository mount, tmpfs writable directory, CPU and memory limits, PID limit, dropped Linux capabilities, `no-new-privileges`, and an in-container timeout wrapper.
- Confirmed host environment variables are not explicitly passed through and no user secret paths are mounted.
- Confirmed unsafe shell chaining and network-oriented commands are rejected by tests.

## Docker live-test status

`docker --version` failed because Docker is not available on PATH in this environment. Live Docker execution is not verified.

Run this later from the repository root after Docker Desktop is installed:

```sh
node packages/cli/dist/index.js verify-safe-run examples/basic-node-bug/reprogate.yml --yes --timeout 60 --cpus 1 --memory 512m
```

In this environment, the same command printed the intended Docker command and then failed safely with: `Docker is unavailable. ReproGate will not fall back to host execution.`

## Known limitations

- Only schema version `0.1` is supported.
- Safe-run is intentionally conservative and rejects shell control operators rather than supporting complex scripts.
- Live Docker safe-run execution still needs verification on a Docker-enabled machine.
- AI adapters are mock-only and advisory; no real provider is implemented.
- Dashboard generation is static and local-only.
- `reprogate create` is interactive and was verified by typecheck/build, not by an automated terminal interaction in this audit.
