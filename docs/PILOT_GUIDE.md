# Pilot Guide

Start with a small pilot on one repository.

1. Add the issue template and GitHub Action.
2. Watch five to ten incoming reports.
3. Note which fields are confusing.
4. Adjust maintainer docs and examples.
5. Decide whether to require ReproGate manifests for specific report types.

Pilot success should be measured by reduced clarification loops and better reproduction quality, not by claims about AI authorship.
